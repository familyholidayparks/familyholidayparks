# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this project is

A static website that lists and ranks family holiday parks across Australian locations. It is deployed to Cloudflare Pages (from the `public/` directory). All HTML pages are generated by Python scripts that pull from scraped park data, Google reviews, and Claude-scored JSON files stored per location.

There is also a Cloudflare Worker (`api-proxy/`) that handles two POST endpoints used by the live site:
- `/` (default) — captures email leads (name, email, instagram) into Airtable and fires an n8n webhook
- `/request-park` — captures park-request submissions into Airtable

## Environment setup

Requires a `.env` file at the project root with:
```
ANTHROPIC_API_KEY=...
AIRTABLE_TOKEN=...
AIRTABLE_BASE_ID=...
WORKER_URL=https://api-proxy.familyholidayparks.workers.dev
```

Python dependencies (requires Python 3.11+):
```
pip install -r requirements.txt
```

## Key commands

**Generate a single location page:**
```
python generate_page.py "Gold Coast QLD"
```

**Score parks for a location (uses Claude API):**
```
python score_parks.py "Gold Coast QLD" --auto
```

**Full overnight build (all locations with approved-parks.json):**
```
python build_all.py
```
This scores, generates pages, generates the leaderboard, then does `git add -A && git commit && git push`.

**Regenerate the homepage:**
```
python generate_homepage.py --publish
```

**Deploy the Cloudflare Worker:**
```
cd api-proxy
npm install
npm run set-secret   # reads AIRTABLE_TOKEN from ../.env and sets it as a Wrangler secret
npm run deploy
```

**Local worker dev:**
```
cd api-proxy && npm run dev
```

**Airtable one-time setup (creates the leads table):**
```
node setup.js
```

## Architecture

### Data pipeline

Each location has its own directory under `locations/<state-abbrev>/<slug>/`, e.g. `locations/qld/gold-coast/`. The pipeline runs in this order:

1. **Scrape** — `scrape_parks.py` fetches raw park listings from Google Maps (via Apify) and writes them to `review-data/<location>-raw-parks.json`
2. **Approve** — `preapprove.py` filters to qualifying parks and writes `locations/<state>/<slug>/approved-parks.json`
3. **Score** — `score_parks.py` calls Claude to generate family-focused scores and writes `locations/<state>/<slug>/scores.json`
4. **Generate** — `generate_page.py` reads scores.json, approved-parks.json, FAQs, hero text, etc. and emits `public/<slug>-<state-name>.html`

`build_all.py` orchestrates steps 3–4 across all locations with an existing `approved-parks.json`, using 3 parallel workers.

### Per-location directory contents

```
locations/<state>/<slug>/
  approved-parks.json     # curated list of qualifying parks (input to scoring)
  scores.json             # Claude-generated family scores (output of score_parks.py)
  config.json             # per-location overrides (hero text, force-include parks)
  hero-tagline.txt        # hero section tagline
  hero-intro.txt          # hero section intro paragraph
  destination-summary.txt # prose overview of the destination
  faq.json                # structured FAQ content
  local-knowledge.txt     # insider tips content
  photos.json             # hero image URLs
  prices.json             # price data per park
  websites.json           # park website overrides
  whitelist.json          # parks forced into inclusion
  reviews/                # per-park review JSON files
```

### Master data files

- `locations.csv` — master list of all locations with `location`, `state`, `slug` columns; `build_all.py` and `generate_homepage.py` read this
- `holiday_parks_master.csv` — enriched master CSV of all park records
- `park-whitelist.json` — global park whitelist (flat `{ "Park Name": true }` format)
- `locations-config.json` — per-location hero config (headline, tagline, intro, force_include)

### Deployment

The `public/` directory is automatically deployed by Cloudflare Pages on every `git push` to `main` (configured via `wrangler.jsonc`). There is no build step — the HTML files are committed directly.

The API worker (`api-proxy/`) is deployed separately via `npm run deploy` from the `api-proxy/` directory. CORS is locked to `familyholidayparks.com.au`.

### Worker secrets and vars

The worker reads from Cloudflare environment bindings set in `api-proxy/wrangler.toml`:
- `AIRTABLE_TOKEN` — set as a Wrangler secret (not in wrangler.toml)
- `AIRTABLE_BASE_ID`, `AIRTABLE_TABLE_ID`, `AIRTABLE_PARK_REQUESTS_TABLE_ID` — set as `[vars]`
